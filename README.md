# SLR-parser

Реализация синтаксического анализатора SLR(1) для арифметического языка
---


##  Требования

- **CMake ≥ 3.16**
- **C++17** (g++ или clang++)
- **FLEX** — для лексера
---

## Структура


- `lexer/` — лексический анализатор (Flex)
- `parser/`
- `parser.h / parser.cpp` — реализация алгоритма SLR
- `slrbuild.cpp` — описание грамматики и таблиц ACTION/GOTO
- `outputstates.*` — вывод шагов работы парсера (стек, вход, действие)
- `tests/` — тесты для корректных и ошибочных выражений
- `CMakeLists.txt` — система сборки проекта
- `README.md` — описание проекта
---

## Cборка

```bash
mkdir build
cd build
cmake ..
cmake --build .
```

После сборки появятся бинарники:

| Бинарник                    | Назначение                                      |
|---------------------------- |--------------------------------------------------|
| `./slr_parser`              | для запуска программы                            |
| `./slr_tests`               | для запуска тестов                               |
---

## Результат работы

Вывод происходит в виде таблицы:
-`текущее содержимое стека`
-`оставшуюся часть входной строки`
-`выполняемое действие (Shift, Reduce, Accept)`

Пример вывода:

```
STACK          | INPUT           | ACTION
------------------------------------------
$              | id+num*$        | Shift
$E             | +num*$          | Reduce
...
```

При обнаружении синтаксической ошибки выводится сообщение:
```
SYNTAX ERROR
```


## Арифметический язык
Арифметический язык состоит из сложения, вычитания, умножения, деления и скобок. Операции могут производиться над переменными или же над числами. Имена
переменных формируются только из букв латинского алфавита (строчные и заглавные)

## Грамматика арифметического языка

Грамматика нашего арифметического языка задаётся следующими правилами:
```
(0) E → E + T
(1) E → E - T
(2) E → T
(3) T → T * F
(4) T → T / F
(5) T → F
(6) F → ( E )
(7) F → ID
(8) F → NUM
```
